# 版本历史

## V3.2 (2025-10-12) - 🔥 关键功能修复

### 重大改进
- **🔥 集成CBC求解器优化** ([CRITICAL_FIX_OPTIMIZER_INTEGRATION.md](docs/CRITICAL_FIX_OPTIMIZER_INTEGRATION.md))
  - 修复了最严重的问题：之前的版本根本没有使用优化求解器
  - 集成 `HeadwayOptimizer` 到主流程，使用PuLP + CBC求解器
  - 优化效果显著：
    - ✅ 发车间隔方差从3071.5降到51.5（降低98.3%）
    - ✅ 均衡性得分从0.911提升到0.998（提升9.6%）
    - ✅ 真正实现了数学优化，而不只是启发式算法

### 执行流程
**修复前**（V3.0-V3.1）:
```
步骤1: 启发式算法生成方案
步骤2: 构建时刻表
步骤3: 格式转换
❌ 没有真正的优化！
```

**修复后**（V3.2）:
```
步骤1: 启发式算法生成初始方案
步骤2: 构建详细时刻表
步骤3: 🔥 发车间隔优化（CBC求解器）← 新增
步骤4: 格式转换
✅ 真正的优化算法！
```

### 技术细节
- 使用线性规划模型优化发车间隔均衡性
- 目标函数：`min Σ |h_i - h_avg|`
- 约束条件：最小/最大间隔、顺序保持、调整范围
- 求解器：CBC (COIN-OR Branch and Cut)
- 求解时间：<1秒（70列车）

## 重要说明

### 为什么V3.2是关键版本？

V3.0和V3.1虽然有完整的架构和代码，但**核心优化功能没有真正工作**：
- `HeadwayOptimizer` 被写好了，但从未被调用
- 程序只使用了简单的启发式算法
- 没有发挥线性规划和CBC求解器的优势

V3.2修复了这个致命问题，**现在才是真正的优化算法**！
