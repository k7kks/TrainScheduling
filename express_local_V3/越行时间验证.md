# 越行时间对称性验证

## 🎯 核心改进

**新的越行逻辑**：让快车通过时间位于慢车停留的正中间

### 原理

```
慢车到达越行站：T
快车通过：      T + Δ（到通间隔）
慢车发车：      T + 2Δ（到通间隔 + 通发间隔）

其中：通发间隔 = 到通间隔 = Δ

这样，快车通过时间正好在慢车停留的中间！
```

### 示例

如果到通间隔是150秒：
```
慢车到达：06:30:00
快车通过：06:32:30（+150秒，到通间隔）
慢车发车：06:35:00（+300秒，总停站）

验证：
- 到通间隔 = 06:32:30 - 06:30:00 = 150秒 ✓
- 通发间隔 = 06:35:00 - 06:32:30 = 150秒 ✓
- 快车在中间 = 150秒 = 150秒 ✓
```

**这正好验证了我们的算法：总停站 = 2 * 到通间隔**

## ✅ 对称性验证

对于任何一次越行：

| 时间点 | 公式 | 说明 |
|--------|------|------|
| 慢车到达 | T | 基准时间 |
| 快车通过 | T + Δ | 到通间隔 |
| 慢车发车 | T + 2Δ | 总停站 |

**到通间隔 = Δ**
**通发间隔 = Δ**
**快车通过位于慢车停留的正中间！** ✓

## 🎨 在运行图中的表现

### 理想效果

```
慢车线：
    ________
   /        \____
  /              \
```

水平部分（停留）：
- 左半部分：到通间隔（等待快车到来）
- 快车通过：正好在中间
- 右半部分：通发间隔（快车通过后等待）

### 验证方法

在运行图显示软件中：
1. 选择一列被越行的慢车
2. 查看越行站的详细时刻
3. 计算：
   - 到通间隔 = 快车通过时间 - 慢车到达时间
   - 通发间隔 = 慢车发车时间 - 快车通过时间
4. **应该相等！**（或者非常接近）

## 📝 注意事项

### 最小间隔限制

如果到通间隔不足120秒，会强制调整：
- 通发间隔 ≥ 120秒
- 总停站时间 ≥ 240秒

这种情况下，快车可能不在正中间，但会尽量接近。

### 最优情况

当到通间隔恰好≥120秒时，越行效果最佳：
- 到通间隔 = 通发间隔
- 快车通过正好在中间
- 停站时间对称
