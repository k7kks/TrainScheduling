## 📝 核心算法说明

### 越行判定条件

当满足以下条件时，会判定需要越行：

```
快车到达时间 < 慢车离站时间 + 最小追踪间隔(120秒)
```

### 越行处理步骤

1. **检测冲突点**：逐站检查快车是否会追上慢车
2. **找越行站**：从冲突点向前查找最近的越行站
3. **调整慢车时刻**：
   - 慢车在越行站停留至少4分钟
   - 确保到通间隔≥120秒
   - 确保通发间隔≥120秒
4. **顺延后续站点**：越行站之后的所有站点时刻顺延

### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 最小追踪间隔 | 120秒 | 前后两车经过同一位置的最小时间差 |
| 最小到通间隔 | 120秒 | 慢车到站到快车通过的最小时间 |
| 最小通发间隔 | 120秒 | 快车通过到慢车发车的最小时间 |
| 越行站最小停站时间 | 240秒 | 慢车在越行站至少停留4分钟 |

## ❓ 常见问题

### Q1: 为什么有的慢车没有被越行？

A: 如果慢车在快车前面出发，且发车间隔足够大，快车就追不上慢车，不需要越行。

### Q2: 能否让一列慢车被多列快车越行？

A: 可以！这叫"二次越行"。调整发车时间和快车数量就能看到。

### Q3: 越行站的选择有什么要求？

A: 越行站需要：
- 有配线（可供慢车停靠等待）
- 有信号设备支持
- 通常是较大的车站

Demo中设定站点3、站点5、站点7可以越行。

### Q4: 如何减少越行次数？

A: 可以：
1. 增大快慢车发车间隔
2. 减少快车比例
3. 调整快车停站方案（减少跳停）
4. 让快车在慢车前面发车
