# 站台码问题修复报告

## 🐛 问题描述

**用户反馈**：
```
Excel结果又出现了站台目的地码和路径编号不匹配的问题，
图中845号车次，它的路径编号是1086，但是它的站台码列表却出现了错误的值，
不符合Path1086的站台码列表
```

**错误现象**：
- 路径1086应该有的站台码：111, 114, 122, 132, 142, 154, 162, 172, 184, 192, 202, 212, 222, 232, 242, 252, 262, 274, 282, 292, 302, 312, 322, 332, 333
- 但Excel中出现了：114, 122, 132, 142, 154, 162, 172, 184, 192, 202, 212, 333, 222, 332, 333, 331, 321...（错乱）

## 🔍 问题根源

**根本原因**：在`_create_route_solution_from_path()`方法中的站台码映射逻辑有误

具体问题：
1. `timetable_builder`生成的`entry.dest_code`是根据车站和方向动态选择的
2. 这个dest_code可能与`path.nodeList`中预定义的站台码不一致
3. 之前的映射策略使用了"按顺序匹配"，会覆盖正确的映射关系
4. 导致RouteSolution中的站台码与path不一致

## ✅ 修复方案

### 修复逻辑

**核心思路**：
- **站台码必须使用path.nodeList中的值**（这是XML中明确定义的）
- **时刻从timetable中获取**（这样才能保留越行调整）
- **通过station_id建立映射关系**（而不是dest_code）

### 修复代码

修改`express_local_V3/main.py`的`_create_route_solution_from_path()`方法：

```python
# 【核心思路】
# 问题：timetable_entries的dest_code可能与path.nodeList不匹配
# 解决：通过station_id建立映射，而不是dest_code
# 站台码必须使用path.nodeList中的值，时刻从timetable中获取

# 建立station_id到时刻表条目的映射
entry_by_station = {}
for entry in timetable_entries:
    if hasattr(entry, 'station_id') and entry.station_id:
        if entry.station_id not in entry_by_station:
            entry_by_station[entry.station_id] = entry

# 建立dest_code到station_id的映射（通过rail_info）
destcode_to_station = {}
for station_id, station in self.rail_info.stationList.items():
    for platform in station.platformList:
        if hasattr(platform, 'dest_code') and platform.dest_code:
            destcode_to_station[platform.dest_code] = station_id

# 遍历path.nodeList，使用path的dest_code作为站台码，但时刻从timetable获取
for i, dest_code in enumerate(path_destcodes):
    # 通过dest_code查找对应的station_id
    station_id = destcode_to_station.get(dest_code)
    
    # 通过station_id查找时刻表条目
    if station_id and station_id in entry_by_station:
        entry = entry_by_station[station_id]
        arrival_time = entry.arrival_time
        departure_time = entry.departure_time
        dwell_time = entry.dwell_time  # 包含越行调整后的停站时间
    else:
        # 回退方案：按顺序使用或估算
        ...
    
    # 【关键】使用path.nodeList中的dest_code
    rs.addStop(
        platform=dest_code,  # 使用path中的dest_code，保证正确
        stop_time=dwell_time,
        perf_level=self.speed_level,
        current_time=arrival_time,
        dep_time=departure_time
    )
```
