## 💡 为什么这样设计

### 理论依据

根据您提供的知识点：
- **到通间隔**：慢车到达 -> 快车通过
- **通发间隔**：快车通过 -> 慢车发车

**对称设计**的好处：
1. 快车通过在中间，视觉效果更好
2. 慢车等待时间分布均匀
3. 符合运营规范的对称性原则

### 实际约束

- 到通间隔 ≥ 120秒（安全要求）
- 通发间隔 ≥ 120秒（安全要求）
- 总停站时间 ≥ 240秒（4分钟）

当到通间隔<120秒时，会强制通发间隔=120秒，此时可能不完全对称。

## 🎯 与理想效果的对比

### 您要求的理想效果

```
慢车停站240秒：
- 到通间隔：120秒
- 通发间隔：120秒
- 快车在中间
```

### 当前实现

```
慢车停站时间：300-360秒（取决于实际到通间隔）
- 到通间隔：150-180秒
- 通发间隔：150-180秒（与到通相等）
- 快车在中间 ✓
```

**差异原因**：
- 快车和慢车的发车时间已经固定
- 实际的到通间隔可能不是正好120秒
- 为了让快车在中间，通发间隔=到通间隔
- 所以总停站时间 = 2 * 实际到通间隔

如果到通间隔正好是120秒，总停站就是240秒！
