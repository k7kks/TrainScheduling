# 越行功能优化说明

## 🎯 优化目标

根据用户反馈的运行图效果，进行以下优化：

### 问题1：越行点位置不够合理

**现象**：越行点不在线路中间，过于靠前或靠后

**优化方案**：
- 改为选择**中间位置**的越行站
- 从所有可用越行站中，选择位于中间的那个
- 让越行点分布更均衡

### 问题2：不必要的越行

**现象**：在没有快车越行的地方，慢车也增加了停站时间

**优化方案**：
- 更精确的冲突判定逻辑
- 只有真正会发生冲突时才越行
- 过滤掉停站时间增加<60秒的情况（可能不需要越行）

## 🔧 具体优化内容

### 优化1：越行点选择（_find_overtaking_point）

**修改前**：
```python
# 从冲突点向前找，返回第一个越行站
for j in range(conflict_point, -1, -1):
    if j in self.overtaking_station_indices:
        return j  # 返回离冲突点最近的
```

**修改后**：
```python
# 找出所有可用的越行站
available_overtaking_stations = []
for j in range(2, conflict_point + 1):
    if j in self.overtaking_station_indices:
        available_overtaking_stations.append(j)

# 选择中间位置的越行站
if len(available_overtaking_stations) > 1:
    mid_index = len(available_overtaking_stations) // 2
    return available_overtaking_stations[mid_index]
```

**效果**：
- 越行点更靠近线路中间
- 分布更均匀

### 优化2：精确的冲突判定

**修改前**：
```python
# 只要快车到达早于慢车发车+120秒，就判定冲突
if express_entry.arrival_time < local_entry.departure_time + self.min_tracking_interval:
    return i  # 立即返回冲突点
```

**修改后**：
```python
# 更精确的判定：
# 1. 快车必须早于慢车发车
# 2. 间隔必须小于最小追踪间隔
if express_entry.arrival_time < local_entry.departure_time:
    time_gap = local_entry.departure_time - express_entry.arrival_time
    if time_gap < self.min_tracking_interval:
        conflict_point = i  # 记录冲突点，继续查找
        break
```

**效果**：
- 减少误判
- 只在真正会冲突时才越行

### 优化3：过滤不必要的越行

**新增逻辑**：
```python
# 计算停站时间增加量
time_increase = new_dwell_time - original_dwell

# 如果增加时间太少（<60秒），可能不需要越行
if time_increase < 60:
    return  # 不应用越行
```

**效果**：
- 避免在不必要的地方增加停站时间
- 只在真正需要避让时才越行
