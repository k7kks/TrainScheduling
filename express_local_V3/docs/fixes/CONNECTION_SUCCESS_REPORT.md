## 🔧 关键修复

### 修复1：时间排序优化

**问题**：原先按`car_info.arr_time`排序，导致车次顺序错误。

**修复**：
```python
# 修改前
trains.sort(key=lambda x: x.car_info.arr_time)

# 修改后
trains.sort(key=lambda x: x.arr_time[0] if x.arr_time else 0)
```

**效果**：车次按首站到达时间正确排序。

### 修复2：勾连策略优化

**问题**：简单的交替连接无法找到最佳勾连。

**修复**：采用贪心算法，为每个车次找折返时间最短的配对：

```python
# 策略1: 为每个上行车找最佳下行车
for up_train in uplink_trains:
    # 找所有可能的下行车
    candidates = []
    for down_train in downlink_trains:
        turnback_time = down_train.arr_time[0] - up_train.dep_time[-1]
        if 30 <= turnback_time <= 1800:  # 30秒到30分钟
            if self._can_connect(up_train, down_train):
                candidates.append((down_train, turnback_time))
    
    # 选择折返时间最短的
    if candidates:
        candidates.sort(key=lambda x: x[1])
        best_down, best_turnback = candidates[0]
        up_train.next_ptr = best_down
        best_down.prev_ptr = up_train
```

**效果**：成功建立26+7+7=40个勾连。

### 修复3：折返时间约束放宽

**问题**：折返时间要求60-900秒太严格，导致部分合理勾连被拒绝。

**修复**：
```python
# 修改前
min_turnback = 60   # 最少60秒
max_turnback = 900  # 最多15分钟

# 修改后
min_turnback = 30    # 最少30秒（放宽）
max_turnback = 1800  # 最多30分钟（放宽）
```

**效果**：更多车次能够成功勾连。

## 🎯 勾连规则总结

### 勾连条件

两个车次可以勾连需满足以下所有条件：

1. **方向相反**：前车上行 ↔ 后车下行
2. **交路相同**：大交路↔大交路，小交路↔小交路
3. **快慢相同**：快车↔快车，慢车↔慢车
4. **折返时间合理**：30秒 ≤ 折返时间 ≤ 30分钟
5. **站台兼容**：终点站和起点站在同一折返区域
6. **未被占用**：前车无后续车次，后车无前序车次

### 勾连策略

1. **按组分类**：快慢×交路 = 4组
2. **按方向分开**：上行、下行
3. **按时间排序**：首站到达时间
4. **贪心匹配**：为每个车次找折返时间最短的配对
5. **递归分配表号**：勾连的车次自动共享表号

## 🚀 与src系统的对比

### 相似点

1. ✅ **Phase2逻辑**：同峰期内勾连
2. ✅ **connect_routes方法**：递归分配表号
3. ✅ **折返约束检查**：站台、时间、方向

### 差异点

| 特性 | src系统 | express_local_V3 |
|------|---------|------------------|
| 混跑勾连 | ✓ 支持 | ✗ 不支持（快慢分开） |
| 移位优化 | ✓ 30次×5秒 | ✗ 未实现 |
| Phase3 | ✓ 跨峰期勾连 | ✗ 仅单峰期 |

### 优势

1. **更简单**：不考虑混跑，逻辑更清晰
2. **更高效**：贪心算法找最优勾连
3. **易扩展**：可以轻松添加新规则
